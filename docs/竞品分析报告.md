# 竞品分析报告 V1

项目：洛克王国异色捕捉追踪工具（轻量化）

版本：V1

日期：2026-05-06

## 1. 分析目的

- 明确本产品在“异色捕捉过程”中的核心价值与边界
- 对比竞品的定位、体验与能力差异，形成可执行的产品策略
- 为项目书的功能范围、版本路线、指标体系提供依据

## 2. 方法与对标维度

### 2.1 竞品范围

- 竞品 1：洛克王国官方微信小程序（官方资产/结果查询）
- 竞品 2：破壳梦（孵蛋预测工具 + 数据贡献 + 论坛社区）

### 2.2 核心对标维度

- 时效性：是否支持会话内实时反馈（当天抓当天看）
- 过程记录：是否具备事件级记录与可追溯能力（会话、日志、校准）
- 规则复杂度承载：对复杂规则的结构化表达与可解释性能力（独立计数线、80 保底、补偿口径等）
- 过程决策支持：将进度与成本信息产品化，支持用户做当下决策（继续/暂停/换目标）
- 传播能力：战报/海报等标准化输出，降低分享成本
- 使用门槛：默认路径是否足够短，是否“打开就能用”
- 信任与权威：数据来源、更新频率、口径透明度与一致性

## 3. 本产品概览

### 3.1 定位

轻量化的异色捕捉追踪工具：以“会话记录 + 即时结算 + 一键分享海报”为核心，把抓捕过程变得可控、可复盘、可分享。

### 3.2 当前核心能力

- 捕获追踪：开始抓捕、记录事件、进度展示、编辑校准
- 结算：球耗成本计算
- 分享：海报生成与保存
- 存档：浏览器本地存储（localStorage），跨 Tab 保持状态

## 4. 竞品 1：洛克王国官方微信小程序

### 4.1 竞品定位

官方资产/结果查询入口，更偏“总览与对账”。

### 4.2 关键体验

- Tab2 默认展示“已拥有精灵”总览列表
- Tab3 点击“异色”后跳转 Tab2 并展示具体异色精灵列表（强引导）
- 存在“数据次日更新，请小洛克稍晚查询”提示（次日同步，非实时）

### 4.3 优势

- 权威与信任：官方背书，天然可信
- 入口与触达：微信生态内低门槛
- 结果查询路径短：异色筛选跳转直达结果列表

### 4.4 局限

- 时效性不足：次日更新不适合承载“当天抓捕过程”的即时反馈与复盘
- 过程不可追溯：更偏结果展示而非事件级记录（每次噩梦枷锁、球耗、偏差补偿等）
- 难承载个体化复杂口径：多条独立计数线（家族/系别/随机互不影响）与补偿逻辑更适合过程工具

### 4.5 我们可借鉴的点

- “按钮触发 → 自动跳转 → 展示筛选结果”的强引导交互，降低路径长度
- 结果列表默认可用、无需理解复杂概念

## 5. 竞品 2：破壳梦 roco-eggs.tsuki-world.com

链接：https://roco-eggs.tsuki-world.com/forum

### 5.1 竞品定位

孵蛋预测垂直工具 + 数据贡献 + 论坛社区。用“输入 → 预测 → 回填”形成数据闭环。

### 5.2 关键体验

- 默认页即“输入栏”（低门槛）
- 输入后自动切到“蛋蛋/结果”页，推送最可能精灵并按概率从高到低排序
- 支持在结果页输入真实结果，便于评估预测效果（PM 视角的闭环）
- 提供“分布星图/列表”展示数据分布，支持筛选
- 提供“贡献数据”入口（结构化表单 + 数据量展示）
- 提供论坛（分类、发帖）承接交流与传播

### 5.3 优势

- 极简默认路径：打开即输入，输入即出结果
- 强反馈呈现：概率排序让用户迅速得到答案
- 数据闭环清晰：回填与贡献数据让产品持续优化
- 工具 + 社区一体化：工具解决任务，社区承接讨论与传播

### 5.4 局限

- 任务聚焦在“孵蛋预测”，非“异色捕捉过程追踪/成本结算/会话日志/保底管理”
- 概率预测类产品的解释成本更高（用户更在意“准不准”），需要持续运营与口径维护

### 5.5 我们可借鉴的点

- 默认态即关键输入态：减少选择题，先让用户开始记录
- 输入完成后自动推进：减少迷路成本
- 结果表达聚焦与排序：把用户最关心的信息放最大
- 建立“可追溯的闭环”：回填/校准是提升可信度与改进速度的关键

### 5.6 数据飞轮

破壳梦具备典型的“数据飞轮（Data Flywheel）/人机协同（Human-in-the-Loop）”特征：

- 数据采集：用户在输入页与贡献页提供结构化特征数据（尺寸、重量等）
- 标签回流：用户在结果页回填“真实孵化结果”，形成近似 Ground Truth 的标签数据
- 评估与迭代：可持续评估预测效果（准确率、Top-K 命中率、校准度等），并用新数据优化推断逻辑
- 正反馈循环：预测更准 → 工具更有用 → 用户更多 → 数据更多 → 预测更准

该飞轮的关键在于：将用户行为自然地转化为数据资产（特征 + 标签），并把回填成本压到最低。

## 7. 关键结论与策略建议

### 7.1 结论

- 官方小程序更适合“权威总账/次日对账”，不适合承担抓捕过程的实时记录
- 破壳梦的核心竞争力在“极简链路 + 强反馈排序 + 数据闭环”，值得我们借鉴其信息架构与闭环设计
- 本产品最优解是成为“过程记账工具”：用实时、可追溯、可结算、可分享的能力补齐竞品空白

### 7.2 策略建议

- 体验策略：默认态即开用，最短路径完成一次会话（开始抓捕 → 记录 ≥ 1 次 → 看结算 → 分享）
- 可信度策略：将规则口径写成产品内可见说明（1.8%/80/补偿）+ 事件日志可追溯 + 支持校准
- 增长策略：把“海报/战报”做成传播素材（模板化、可复用），承接社群扩散
- 迭代策略：优先做“过程闭环”而非“权威对账”，避免与官方正面竞争

### 7.3 AI 产品经理视角补充

- 数据资产化：以“事件日志 + 会话数据 + 回填/校准”沉淀结构化数据，为后续智能化（建议/预测/异常检测）提供数据基础
- 可解释性优先：规则复杂度高时，优先做“口径透明 + 可追溯”的解释链路，再谈模型能力
- 评估框架：建立 Top-K 命中、校准度、用户回填率、纠错率等指标，支撑迭代决策与优化闭环
