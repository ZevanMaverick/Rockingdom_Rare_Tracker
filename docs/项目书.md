# 项目书 V1

项目：洛克王国异色捕捉追踪工具（轻量化）

版本：V1

相关文档：

## 1. 执行摘要

### 1.1 一句话定位

一款轻量化的《洛克王国》异色捕捉追踪工具，用“会话记录 + 即时结算 + 一键海报”把抓捕过程变得可控、可复盘、可分享。

### 1.2 核心价值点

- 实时：抓捕过程中即时记录与反馈，补足官方“数据次日更新”的延迟
- 低门槛：默认路径短，打开就能开始记录
- 可追溯：事件日志 + 可编辑校准，保证长期可用与可解释
- 可决策：球耗成本即时结算，支持预算管理与复盘
- 可传播：战报/海报标准化输出，降低分享成本

### 1.3 产品目标与边界

- 目标：用最低操作成本完成“当日过程记录 → 即时复盘 → 可分享输出”的闭环
- 边界：不做官方权益/活动入口，不做“权威对账”；定位为官方小程序的过程工具补充

### 1.4 当前进展

- Tab1 捕获追踪：开始抓捕、事件记录、进度展示、编辑校准、跨 Tab 状态保持
- Tab3 结算：球耗成本相关能力
- 分享海报：生成图片保存
- 部署：支持 Vercel/Netlify 静态部署

## 2. 背景与机会

### 2.1 用户问题

- 异色捕捉是高频、长周期、强运气波动的过程任务，用户容易出现“漏记、记错、无法复盘、成本不透明”的问题
- 官方小程序更偏资产总览与次日对账，无法承载当天抓捕过程中的实时记录与即时反馈

### 2.2 机会判断

- 过程工具价值：越是强波动（概率/保底/补偿），越需要可追溯的过程记录
- 社群传播价值：异色成果天然适合晒战报，模板化海报能转化为增长杠杆

## 3. 目标用户与核心场景

### 3.1 用户分层

- 新手玩家：需要快速上手、避免算错、减少记录成本
- 重度收集玩家：多目标切换、长周期追踪、需要校准与复盘
- 内容分享型玩家：需要一键输出可分享素材（战报/海报）

### 3.2 核心场景

- 边抓边记：会话内快速记录关键事件
- 当天复盘：成本、进度与关键事件回看
- 多目标切换：切换精灵/家族/系别后仍能正确读取对应计数
- 社群分享：海报一键生成与保存

## 4. 规则口径与产品记录模型

本章节用于对齐“我们记录的是什么、怎么算、怎么解释”。

### 4.1 游戏口径

- 保底独立计数并生效：
  - 各个家族的连续捕捉独立计数，互不影响
  - 各个系别的连续捕捉独立计数，互不影响
  - 随意捕捉也有独立计数的保底，与家族/系别互不影响
- “噩梦枷锁”触发并击败污染精灵后：
  - 约 1.8% 的综合概率出现异色精灵
  - 在保底规则下最多 80 次“噩梦枷锁”事件即可遇到异色精灵
  - 存在额外补偿规则，使实际综合概率提升到 2% 以上

### 4.2 产品口径

- 事件单位：一次“噩梦枷锁”事件是一次记录单位（发生一次，计数/清零/日志变化一次）
- 进度上限：80（封顶显示）
- 记录能力：支持事件日志与“编辑校准”（用于误操作或补录）

### 4.3 可解释性与一致性

- 口径透明：将关键规则（概率/保底/补偿、计数互不影响）在产品内以“说明 + 示例”的方式固定表达
- 可追溯：每次记录形成事件日志，便于复盘与纠错
- 可校准：提供编辑校准能力，保证“工具长期可用”，避免因漏记导致用户放弃使用

## 5. 产品方案

### 5.1 信息架构

- Tab1：捕获追踪（核心）
- Tab2：精灵图鉴（浏览与筛选）
- Tab3：结算（成本/收益相关）

### 5.2 Tab1 捕获追踪

- 默认态：选目标精灵、展示进度、提示下一步操作
- 进行中：突出“当前目标进度”，满足最短路径高频记录
- 记录操作：将关键事件抽象为按钮，降低输入成本
- 操作日志：可回看每次记录（时间 + 类型 + 描述）
- 编辑校准：在需要时一次性修改关键进度数据

备注：为保持轻量与聚焦，界面默认不展示“系别池进度”。

### 5.5 数据闭环设计

- 数据结构：沉淀结构化事件（时间、类型、目标、增量/清零、成本摘要）
- 回填/校准：将“纠错”产品化，形成真实使用数据闭环
- 评估指标：以回填率、纠错率、会话完成率衡量记录口径的可用性与一致性
- 智能化前置条件：先沉淀高质量事件数据与口径稳定，再引入推荐/预测能力

### 5.3 Tab3 结算

- 球耗成本：记录抓捕前后持有量，计算差值作为消耗与成本
- 复盘输出：与 Tab1 的会话与海报形成闭环

### 5.4 分享海报

- 输出口径：当前目标 + 关键进度 + 成本摘要
- 目标：让用户“保存即分享”，降低传播成本

## 6. 核心用户路径

### 6.1 新手最短路径

- 进入 Tab1 → 选择目标精灵 → 开始抓捕 → 记录 ≥ 1 次 → 查看成本与进度 → 分享海报

### 6.2 重度玩家路径

- 进行中记录 → 切换目标 → 读取对应计数 → 必要时编辑校准 → 复盘会话 → 分享

### 6.3 传播路径

- 会话记录 → 生成海报 → 社群分享 → 回流继续记录

## 7. 竞品结论与差异化策略

### 7.1 我们与官方小程序的关系

- 官方小程序适合“权威总账/次日对账”
- 我们适合“过程记账/当天决策”

### 7.2 我们从破壳梦学习

- 默认态即关键输入态，输入完成后自动推进
- 结果表达聚焦与排序，建立“回填/校准”的数据闭环

## 8. 指标体系与验收标准

### 8.1 北极星指标

- 日均有效会话数（开始抓捕并记录≥1次的会话）

### 8.2 漏斗指标

- 访问 → 开始抓捕 → 记录≥1次 → 查看结算 → 生成海报 → 次日留存

### 8.3 质量指标

- 记录成功率、海报生成成功率、异常/崩溃率

### 8.4 增长指标

- 分享率：生成海报/保存海报占有效会话比例
- 回流率：分享后 24 小时内回流继续记录比例

## 9. 版本规划

### 9.1 MVP

- 会话记录 + 成本结算 + 海报分享 + 本地存档

### 9.2 V1

- 历史会话列表与筛选
- 导出/导入存档（JSON），解决换设备/换浏览器数据丢失
- 模板化海报（简洁版/数据版）

### 9.3 V2

- 云同步/多端（账号体系与冲突合并策略）

### 9.4 智能化路线

### 9.5 内容更新与赛季适配

- 内容更新机制：跟随官方赛季/版本迭代更新精灵与异色收集目标，形成稳定的更新节奏与变更记录
- UI 适配规划：新增异色精灵时同步调整图鉴与筛选结构，突出“本赛季新增异色/近期新增”入口，降低用户查找成本
- 质量与一致性：为数据字段与展示口径做版本化管理，确保历史会话与新赛季内容兼容

### 9.6 数据洞察与概率估计

- 用户规模化数据采集前提：需要云端存档或显式用户授权的匿名数据上报；默认本地存储不做跨用户聚合
- 指标产出：基于会话与事件数据，统计不同精灵的平均出异色概率与分布区间，并提供时间窗（赛季/版本）维度对比
- 方法论约束：明确样本量门槛、选择偏差与口径差异，必要时输出置信区间与免责声明，避免把估计值误读为官方概率

- 智能总结：自动生成会话摘要与“关键事件时间线”
- 目标建议：基于历史会话与成本，给出“继续/暂停/换目标”的提示（以规则可解释为前提）
- 风险提示：识别异常记录（连续高频误触、进度异常跳变），提醒用户校准

## 10. 技术与部署

- 前端：React + Vite + TypeScript
- 存储：localStorage（默认），后续可拓展云同步
- 部署：静态站点部署（Vercel/Netlify），构建产物 `dist/`

## 11. 风险与对策

- 规则口径争议：在产品内提供口径说明与版本记录
- 误操作/漏记：提供日志回看与编辑校准
- 数据丢失（换设备）：提供导出/导入（V1）或云同步（V2）
- 合规与素材：明确素材来源与免责声明
